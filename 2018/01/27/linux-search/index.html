Linux中的搜索命令

发表于 2018-01-27 | 分类于 Linux

locate

用于搜索指定的文件

  • 调用方法 locate {file name}
  • /var/lib/mlocate 是locate查询到数据库
  • updatedb更新数据库
  • 查询遵循/etc/updatedb.conf的配置来查询
  • 配置默认不搜索一些文件夹和文件系统，来减少误解

whereis

用户查询命令以及其帮助文档的位置。

  • 调用方法 whereis {command}
  • options
    -m 只查找manual
    -b 只查找binary
  • 命令是干什么的 whatis {command}

whichis

查找命令的位置，并显示其别名，如果有的话
whichis类似于whereis

  • 对于shell内置的命令，两者都找不到，因为没有可执行文件
  • 两者都是在PATH路径里面寻找命令

find

find是Linux中最强大的搜索命令

  • 使用方法：find [搜索范围] [搜索条件]
  • find默认情况下是完全匹配的，搜寻的条件和查询结果一致，模糊查询需要使用通配符
  • 通配符
    1. * 匹配任意多个字符
    2. ? 匹配任意一个字符
    3. [] 匹配任意一个出现在括号里的字符
  • options:
    1. -name 按文件名
    2. -iname 按文件名，不区分大小写
    3. -user 按文件所有者
    4. -nouser 查找没有所有者的文件
    5. -atime -ctime -mtime {[+-]day} 查找对应时间的相应操作的文件
    6. -size {[+-]size} 查找[大于小于]正好等于size大小的文件
    7. -inum {inode} 按照inode号来查找文件
    8. {条件1} -a {条件2} 同时满足条件1和条件2
    9. {条件1} -o {条件2} 满足条件1或条件2
    10. -exec command {} \; 将find的结果替换{}然后执行command命令,\;代表一条一条结果执行command
    11. -exec command {} + 类似上一条，不过+代表全部结果执行一条命令

      grep

      在文件中，查找符合要求的字符串
  • 使用方法grep [options] {file name}
  • options：
    1. -v 取反，选择不包含关键字的字符串
    2. -i 忽略大小写
  • grep是包含匹配，区别于find
  • grep可以使用正则表达式来匹配
  • 匹配内容使用通配符，匹配字符串（文件内容）使用正则表达式
