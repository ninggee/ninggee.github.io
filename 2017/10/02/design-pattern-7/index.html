Pattern 7. Prototype模式

发表于 2017-10-02 | 分类于 设计模式学习

定义

Specify the kinds of objects to create using a prototypical instance,and create new objects by copying this prototype.

使用一个原型的实例指定要创建的对象的种类，然后通过复制这个原型来创建对象。

实际上原型模式也是一个比较简单的模式，学习到现在，简单的模式一共有三个：

  • 单例模式
  • 迭代器模式
  • 原型模式

对于原型模式来说，重要的就是创建新的对象了，可以看到，重点就是通过复制的方式生成对象。

如何工作

对于原型模式如何工作，它的定义已经讲的很清楚了，就是：使用复制来生成对象。
对于JAVA来说，复制对象是一件很简单的事情，首先，Object对象中就存在一个clone方法，其次，还有一个Cloneable接口，标识了一个对象是可以拷贝的，所以，只要我们在声明类的时候，实现这个接口，并覆写clone方法就可以实现原型模式了。

原型模式中的角色

  1. Prototype: 原型对象负责定义使用复制来生成新对象的方法（接口），同时在类中可以加入对子类的约束
  2. Concrete Prototype：实现原型定义的生成新对象的接口，同时实现约束。

原型模式的优点

  1. 性能好
    原型模式（clone方法）是对内存二进制流的复制，比直接new一个对象的性能要好的多。
  2. 不会执行构造函数
  3. 可以生成一些难以通过new生成的对象
    比如要实现对用户自己画的一个图的复制，通过new一个新对象然后再修改值得方法是很复杂，很难实现的，通过原型模式，就很容易实现这样的功能
  4. 解耦框架和生成的实例
    可以让生成实例的框架不依赖具体的类，使用事先注册的对象，对其进行复制而不是使用类名来生成对象可以实现解耦。

原型模式的应用场景

  1. 类的初始化需要消耗大量资源的时候
  2. 对性能和安全要求比较高的情况
  3. 不容易通过new生成对象的时候
  4. 多个修改者可能修改一个对象的时候，可以避免相互影响

原型模式的注意点

  1. 构造函数不会被执行
  2. 深拷贝与浅拷贝的问题
  3. final关键字影响深拷贝的问题
